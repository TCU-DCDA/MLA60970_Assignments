# GitHub Setup and Classroom Access Instructions

## Overview
This guide will help you set up your GitHub account and access GitHub Classroom assignments for MLA60970. GitHub is essential for this course as you'll use it for version control, collaboration, and publishing your websites.

## Part 1: Creating Your GitHub Account

### Step 1: Sign Up for GitHub
1. Go to [github.com](https://github.com)
2. Click **"Sign up"** in the top right corner
3. Choose a professional username that you'd be comfortable using professionally
   - **Good examples:** `john-smith`, `jsmith-dev`, `johnsmith2024`
   - **Avoid:** `partyguy123`, `ilovecats`, or anything unprofessional
4. Use your school email address for the account
5. Create a strong password
6. Complete the verification process

### Step 2: Verify Your Email
1. Check your email for a verification message from GitHub
2. Click the verification link
3. This step is **required** before you can access classroom assignments

### Step 3: Set Up Your Profile (Recommended)
1. Add a profile picture (professional headshot or appropriate image)
2. Add your real name to your profile
3. Consider adding a brief bio mentioning you're a student
4. This helps with professional networking and makes your work more credible

## Part 2: Understanding GitHub Basics

### Key Concepts You'll Need
- **Repository (Repo):** A project folder that contains all your files and tracks changes
- **Commit:** Saving a version of your work with a description of what changed
- **Push:** Uploading your local changes to GitHub's servers
- **Pull:** Downloading the latest version from GitHub to your computer
- **GitHub Pages:** Free web hosting service for static websites

### Why We Use GitHub
- **Version Control:** Track changes and revert if needed
- **Backup:** Your work is safely stored in the cloud
- **Collaboration:** Share work with instructors and peers
- **Professional Portfolio:** Showcase your projects to potential employers
- **Industry Standard:** Used by virtually every tech company

## Part 3: Accessing GitHub Classroom Assignments

### What is GitHub Classroom?
GitHub Classroom is a tool that allows instructors to distribute assignments through GitHub repositories. Each student gets their own copy of the assignment to work on.

### Step 1: Getting the Assignment Link
1. Your instructor will provide a GitHub Classroom assignment link
2. The link will look something like: `https://classroom.github.com/a/abc123xyz`
3. **Important:** Only click this link AFTER you've created and verified your GitHub account

### Step 2: Accepting Your First Assignment
1. Click the assignment link provided by your instructor
2. You'll be redirected to GitHub Classroom
3. **Sign in** with your GitHub account if prompted
4. **Select your name** from the class roster
   - If your name isn't listed, contact your instructor immediately
   - Make sure you're using the email address associated with the class
5. Click **"Accept this assignment"**
6. Wait for GitHub to create your personal repository (this may take a few minutes)

### Step 3: Accessing Your Assignment Repository
1. After accepting, you'll see a link to your assignment repository
2. Click the link to view your assignment
3. **Bookmark this repository** - you'll return to it frequently
4. The repository name will typically include your username: `assignment-name-yourusername`

## Part 4: Working with Your Assignment Repository

### Viewing Your Assignment
1. Your assignment repository contains all the files and instructions you need
2. Look for files like:
   - `README.md` - Assignment instructions and overview
   - Template files for your work
   - Any starter code or resources

### Making Changes (Basic Web Interface)
1. **To edit a file:** Click the file name, then click the pencil icon (Edit)
2. **Make your changes** in the web editor
3. **Save your work:** Scroll down to "Commit changes"
   - Add a brief description of what you changed
   - Click "Commit changes"

### Downloading Your Repository (Advanced)
If you prefer to work locally on your computer:
1. Click the green **"Code"** button
2. Select **"Download ZIP"**
3. Extract the files to your computer
4. Work on them locally, then upload changed files back to GitHub

## Part 5: Publishing with GitHub Pages

### Enabling GitHub Pages for Your Projects
1. In your repository, go to **Settings** (top navigation)
2. Scroll down to **"Pages"** in the left sidebar
3. Under **"Source"**, select **"Deploy from a branch"**
4. Choose **"main"** branch and **"/ (root)"**
5. Click **"Save"**
6. Your website will be available at: `https://yourusername.github.io/repository-name/`

### Important Notes for GitHub Pages
- Your repository must be **public** (free GitHub accounts)
- Your main page must be named `index.html`
- Changes may take 5-10 minutes to appear on your live site
- Always test your links and images after publishing

## Troubleshooting Common Issues

### "Assignment link doesn't work"
- Make sure you've verified your email address
- Try logging out of GitHub and logging back in
- Clear your browser cache and cookies
- Contact instructor if problems persist

### "Can't find my name in the roster"
- Check that you're using the correct email address
- Make sure you're logged into the right GitHub account
- Contact your instructor to verify roster information

### "GitHub Pages isn't working"
- Ensure your repository is public
- Check that your main file is named `index.html`
- Wait 10-15 minutes for changes to propagate
- Verify GitHub Pages is enabled in repository settings

### "Lost my repository link"
- Go to [github.com](https://github.com) and sign in
- Your repositories are listed on your homepage
- Look for repositories with your course assignment names
